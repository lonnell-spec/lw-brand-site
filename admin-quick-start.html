QUICK START GUIDE.

How ldw.build is built, where everything lives, and how to update it without breaking anything.

The Stack

HOW THE SITE WORKS

ldw.build is a static HTML site — no framework, no build step, no CMS login. Every page is a single HTML file hosted on Vercel, connected to a GitHub repo. When a file is pushed to the main branch of the repo, Vercel automatically deploys it. That's the entire pipeline.

Dynamic content (Instagram grid, newsletter signups, speaking inquiries) runs through Supabase. Notion is the source of truth for admin documentation.

VERCEL
Hosts the site. Auto-deploys when GitHub main branch is updated. Zero configuration needed after initial setup.
GITHUB
Repo: lonnell-spec/lw-brand-site. All HTML files live here. Push to main = live in ~30 seconds.
SUPABASE
Handles form submissions, Instagram grid data, newsletter sync, and Edge Functions for ConvertKit.
NOTION
Source of truth for admin page content. Sync admin pages to regenerate from Notion data.
Deployment

HOW TO UPDATE THE SITE

  1. 01
    Get the updated file — Claude generates the updated HTML file in the conversation and outputs it to downloads.
  2. 02
    Drop into the local repo — Replace the existing file in your local copy of lonnell-spec/lw-brand-site.
  3. 03
    Commit and push — git add . → git commit -m "description" → git push. Vercel picks it up automatically.
  4. 04
    Verify live — ldw.build usually reflects changes within 30–60 seconds of push.
Shortcut
Claude can also push directly to GitHub via API. When working in a Claude session, say "push this to GitHub" and it will commit and deploy without you needing to touch the repo.
Content Updates

WHAT JADYN CAN UPDATE

Content that lives in Notion databases can be updated without touching code. After updating in Notion, trigger a sync to push changes to Supabase, which the site reads from.

  1. ◆
    Instagram Grid — Update the LDW Site — Instagram Notion database. Max 4 posts displayed.
  2. ◆
    Photo Gallery — Update the LDW Site — Photos Notion database. Sync to push live.
  3. ◆
    Content Hub — Add sermon clips, podcast episodes, and teaching content to the Notion Content Hub database.
Important
Only update files you understand. If something looks broken after an update, do not push another change — contact Claude with the specific error and the original file will be restored.
